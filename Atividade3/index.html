O termo Progressive Web Apps (PWA) foi cunhado em 2015 pelo designer Frances Berriman e pelo engenheiro do Google Chrome Alex Russel para se referir a aplicações que utilizam das novas funcionalidades de Web Browsers modernos para oferecer ao usuário uma experiência similar a uma aplicação nativa (TANDEL & JAMADAR, 2018), um motivo que trouxe o PWA a tona foi a percepção da Google de que a maioria dos sites são acessados por dispositivos móveis (ANTUNES, 2019). O PWA tem ganhado mais destaque devido a iniciativas de empresas como Google e Microsoft (ANTUNES, 2019). Uma vantagem de PWA’s sobre aplicações nativas disponibilizadas em lojas de aplicativos é que PWS’s podem ser adicionados aos dispositivos apenas como um atalho que inicia a aplicação sem a necessidade de instala-lo e pode ser compartilhado como um simples link aumentando o alcance (RF1; ANTUNES, 2019; CECONI, 2020), o que é uma ótima característica considerando o declínio nas instalações de aplicativos em lojas de aplicativos apontado por Comscore (2017) citado por Ceconi (2020).

CARACTERÍSTICAS DE PROGRESSIVE WEB APPS

PWA é uma metodologia de desenvolvimento que busca oferecer aos usuários de aplicações nativas a mesma experiência que eles já tem porém em uma aplicação WEB (ANTUNES, 2019), por isso um PWA pode enviar push-notifications, funcionar offline, ser adicionado a tela inicial ou área de trabalho do dispositivo, dão suporte à splash screens e permitem a execução de código em segundo plano (TANDEL & JAMADAR, 2018).

A Google define algumas características para PWA’s que devem ser seguidas a fim de possibilitar que um PWA entregue a mesma experiência de aplicação nativa à um usuário (ANTUNES, 2019), estas são (TANDEL & JAMADAR, 2018; ANTUNES, 2019):

  • Progressivo: Um progressive web app deve estar acessível a qualquer um, independente do navegador usado ou do local onde se encontra no mundo.
  • Responsivo: Tem a capacidade de se adequar a qualquer tipo de dispositivo que existem e que virão a existir.
  • Experiência de aplicativo: Pelo estilo de navegação e interação serem os mesmos de aplicações nativas, o usuário tem a sensação de estar de fato usando um aplicativo.
  • Independente de conectividade: Usando servisse workers PWA’s podem funcionar normalmente offline ou mesmo em conexões lentas.
  • Instalável: Um usuário pode adicionar um PWA à sua tela principal ou área de trabalho e depois executá-lo como uma aplicação normal, sem precisar acessar uma loja de aplicativos.
  • Descobrível: Devido aos servisse workers e o manifesto da W3C PWA’s podem ser facilmente encontrados por motores de busca.
  • Envolvente: Funcionalidades como push notifications e a não necessidade de uma loja de aplicativos leva o usuário a se envolver mais com o uso da aplicação.
  • Seguro: Por serem servidos via HTTPS, há a garantia que os dados não serão adulterados.
  • Atual: PWA’s estão sempre atualizados devido ao processo de atualização dos servisse workers.

    • TECNOLOGIAS PWA

    Os Progressive Web Apps são possíveis principalmente devido a duas tecnologias, os servisse workers e o manifesto aplicação.

    Service workers

    Um servisse worker nada mais é que um script que é executado separadamente da página e em segundo plano, este recurso permite experiencias offlines e dá ao desenvolvedor total controle sobre a experiencia do usuário (GAUNT, 2020). Estes scripts não podem manipular a DOM (Document Object Model), ao invés disso se comunicam com a página que podem manipulá-la respondendo mensagens enviadas por esta (GAUNT, 2020).

    Manifesto

    O Web App Manifest é arquivo no formato JSON que prove as informações da aplicação, algumas informações que esse manifesto prove é o nome da aplicação, descrição, ícones, língua, orientação da aplicação dentre outras (MDN, 2020).

    Frameworks que possibilitam o desenvolvimento de PWA’s

    Os principais frameworks front-end suportam o desenvolvimento de PWA’s, alguns exemplos desses frameworks são:

    • Angular
    • Ionic
    • ReactJs
    • Vue
    Web Assembly

    Web Assembly é uma tecnologia dos navegadores modernos que permite a execução de código compilado voltado a instruções de baixo nível tendo como linguagens de programação C, C++, Rust, entre outras (MDN, 2020). Também é possível desenvolver PWA’s com tecnologias Web Assembly, um exemplo de tecnologia que funciona em Web Assembly e tem suporte a PWA é o Blazor que é desenvolvido pela Microsoft e permite a execução de código .NET (C# e F#) no lado do cliente. Outro framework que possibilita isso é o Go App (Go lang).

    EMPRESAS QUE USAM PWA

    Alguns exemplos de casos de uso de PWA são o Twitter, OLX e o Flipkart. Com a abordagem do PWA o Twitter lançou o Twitter Lite que é consideravelmente mais leve e rápido que o aplicativo comum e trouxe um aumento de 75% nos tweets enviados e diminuição de 20% na taxa de rejeição (ANTUNES, 2019). Já a OLX ao trazer uma versão em PWA de seu APP obteve um aumento de 250% no engajamento, redução de 80% na taxa de rejeição e um aumento de 146% em cliques em anúncios (ANTUNES, 2019). A Flipkart ao lançar um PWA voltado ao mercado indiano devido a baixa qualidade da rede que a maior parte dos usuários tinham acesso, gerou um reengajamento maior de 40%, obteve uma redução de 3x o uso de dados e por consequência aumentou as conversões em 70% (ANTUNES, 2019).

    CONCLUSÃO

    A tecnologia de Progressive Web Apps vem ganhando cada vez mais destaque, principalmente com o apoio de grandes empresas como Google e Microsoft, além dos diversos casos de sucesso de diversas outras empresas. Ao optar por entregar uma solução PWA os recursos necessários para o desenvolvimento serão menores uma vez que os principais frameworks front-end oferecem suporte à esta modalidade de desenvolvimento possibilitando que apenas um código fonte seja usado tanto para a aplicação WEB quanto para o “aplicativo” que o usuário poderá adicionar à sua área de trabalho seja mobile ou desktop.
